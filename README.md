# Ultrametric dynamic programming for minimum Riesz s-energy subset selection

This bundle contains a small, self-contained Python reference implementation of the exact dynamic programming algorithm for **minimum Riesz `s`-energy subset selection on finite ultrametric spaces**.

The implementation follows the paper notation closely:

- `u` = a node of the rooted binary ultrametric tree
- `Δ_u` = the common cross-subtree distance at node `u`
- `F_u(t)` = the minimum Riesz `s`-energy among all `t`-subsets below node `u`

The recurrence implemented in the code is

```text
F_u(t) = min_{t_v + t_w = t} [ F_v(t_v) + F_w(t_w) + t_v t_w Δ_u^{-s} ]
```

for an internal node `u` with children `v` and `w`.

## Files

- `ultrametric_dp.py` — the main implementation
- `validate_ultrametric_dp.py` — deterministic validation against brute-force enumeration
- `validation_results.json` — generated by the validation script
- `README.md` — this file

## Requirements

Python 3.10+ is sufficient. The code uses only the Python standard library.

## Quick start

### 1. Run the primate example from the paper

```bash
python3 ultrametric_dp.py --example primate --s 1.0 --k 2 --bruteforce
```

This uses the rooted ultrametric tree with four taxa:

- 0 = Human
- 1 = Chimpanzee
- 2 = Gorilla
- 3 = Orangutan

and checks the DP answer against brute force.

### 2. Run one deterministic random example

```bash
python3 ultrametric_dp.py --example random --n 8 --seed 7 --s 2.0 --k 4 --bruteforce
```

### 3. Run the full validation batch

```bash
python3 validate_ultrametric_dp.py
```

This writes `validation_results.json` and prints a summary.

## Validation summary

The current deterministic validation batch checks:

- 12 primate-example instances (`s ∈ {0.5,1,2,3}` and `k ∈ {2,3,4}`), and

## Notes on the implementation

### Tree model

The ultrametric space is represented by a rooted binary tree. Distances are defined by

```text
d(x,y) = 2 * height(lca(x,y))
```

where `height(lca(x,y))` is the merge height of the least common ancestor.

### Energy convention

The code uses the **unordered-pair** Riesz energy

```text
E_s(S) = sum_{ {x,y} subseteq S } d(x,y)^(-s)
```

which is the convention used in the paper.

### Why the DP is exact on ultrametrics

At an internal node `u`, every cross-child pair has the same distance `Δ_u`, so the full cross-term is simply

```text
t_v t_w Δ_u^{-s}
```

This is what makes the DP exact. By contrast, for general tree metrics or for ordered points on the line, the corresponding interaction depends on more than just child cardinalities.
